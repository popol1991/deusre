Instructions for Relevance Assessing Table Search in Electrophysiology

Introduction
Thank you for participating in the evaluation of a new search engine designed to find data tables about neuroscience research. If you want to know more about the project, see our homepage. This web page provides a brief description of the task that you are asked to perform. At the bottom of the page there is a link that will take you to the search engine.

The search engine contains about thirty thousand tables extracted from NeuroElectro.org and journals published by Elsevier. NeuroElectro provided curated versions of tables published in a variety of sources. The Elsevier tables were published in Elsevier neuroscience journals.

Your task consists of two parts: (i) Enter a query, and (ii) Assess the quality of each table that is returned.

1. Entering Queries
Queries consist of words and optional filters. A few example queries are shown below.

Query Filters Description
occipital cortex A typical unstructured query
occipital cortex Value: 10, 50 Prefer columns that contain values between 10 and 50.
occipital cortex Value: 10, 50; Error: 0, 2 Also prefer columns that contain error values between 0 and 2.

Numerical filters can be added by clicking the Add Filter button. There are 11 filters. Value, Error, Magnitude, and p-value apply to values of cells. Integer-Ratio, Real-Ratio, Mean, StdDev, Range, Accuracy, and Magnitude apply to column statistics.

Please enter queries about topics that you know reasonably well. You are free to use the filters if they seem useful, or ignore them, if they do not.

2. Performing Quality Assessments
The most important part of the task is your assessment of the quality of each table returned by the search engine. Your assessments will be used to guide further development of the search engine, so it is important that they are as consistent and unbiased as possible.

To save screen space, the search engine only shows a few of top rows for each table. To show the complete table, click the blue link on the upper right of each table. Click on the same link to hide an expanded table.

To avoid causing bias about which results are best, search results are shown in a random order on the results page. This may feel awkward to you. Most people are unconsciously accustomed to the relevance-based ranking used by commercial search engines, in which the best results are shown first. Please try to remember that these results are in a random order. The best results may be located anywhere on the page. Because results are shown in a random order, it may be a little difficult to quickly get a sense of what the search engine found. When the results are returned, please examine the first 5 tables to get a sense of what the search engine found, before you begin labeling tables. This may help you to label tables more consistently.

Please assign one of the following labels to each table.

  • Key Result: This table is dedicated to the topic; authoritative and comprehensive, it is worthy of being a top result in a table search engine.
  • Highly Relevant: The content of this table provides substantial information on the topic.
  • Relevant: The content of this table provides some useful information on the topic, although it may be minimal.
  • Not Relevant: The content of this table does not provide useful information on the topic, but may provide useful information on other topics, including other interpretations of the same query.
  • Broken: This table is difficult to judge because it is broken or not readable.

After examining a table, please click one of the relevance labels. When you have assessed all of the tables returned for the query, please click the SAVE button at the bottom of the page to save your assessments.

Number of Queries
We need as many queries and assessments as you are willing to provide. We would like to ask you to enter at least 30 queries, and explore (if available) at least 50 results from each query. However, you may stop at any time without consequences to yourself or our research, as long as you save the results after you have performed them.

If you have any further questions about this project, please contact Anita de Waard at Elsevier (a.dewaard@elsevier.com). If you have any further questions about the query assessment system, please contact Kyle Gao at CMU (yingkaig@cs.cmu.edu).

Thank you for your help, we sincerely appreciate it!
The CMU/Elsevier Data Search Team
